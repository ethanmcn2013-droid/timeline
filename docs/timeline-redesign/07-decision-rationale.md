# Design direction decision rationale

Decision status: pending explicit founder selection
Allowed outcomes: Option A, Option B, Option C, or a hybrid naming exact components
Production status: unchanged

## Why this is a direction decision

The current production audit found capable but fragmented surfaces. Owner curation, public plan, shared update, and detail do not yet operate as one guaranteed public fact set. The Phase 1 lab therefore holds data, publication, vocabulary, visibility, and state behaviour constant while varying the experience architecture.

The decision is not which colour treatment looks best. It is which structural bias should govern Timeline when clarity, owner maintenance, public trust, and shareability compete.

## Neutral comparison

| Decision lens | A. Quiet Direction Ledger | B. Editorial Plan Room | C. Signal Horizon |
| --- | --- | --- | --- |
| Primary optimisation | Fast repeated curation and direct scanning. | Purpose, change explanation, and forwarding. | Relative direction and brand-distinctive spatial hierarchy. |
| Dominant structure | Ruled vertical ledger. | Edited briefing with decision margin. | Weighted horizon with contextual rail. |
| Owner advantage | Compact comparison and controls near the selected row. | Public wording and editorial judgement can be considered together. | Stable selected-item rail on wide screens. |
| Public advantage | Predictable ladder with minimal reading overhead. | Strong lead direction and human narrative. | Immediate near-to-later relationship without a Gantt. |
| Update advantage | Concise change, reason, and next-step ledger. | Most article-like and forwardable composition. | Clearest before-and-after movement grammar. |
| Main cost | Less expressive narrative hierarchy. | More vertical space and editorial discipline required. | More responsive and implementation complexity. |
| Primary failure risk | Becoming a plain task list. | Becoming a long report or marketing page. | Implying false schedule precision or becoming a decorative diagram. |

No row in this table is a selection or a quality score.

## Evidence that should drive selection

1. Ten-second orientation on an anonymous public link.
2. Time to complete a realistic owner edit, move with reason, reorder, refusal, recovery, and publish sequence.
3. Accuracy of public, update, and detail facts after the owner change.
4. Comprehension of a forwarded update without prior product context.
5. Refusal reason and date comprehension without shame or alert styling.
6. Mobile reading order, touch access, keyboard access, reduced motion, print, and dense-data behaviour.
7. Maintenance cost when mapped to the real production architecture.

Screenshots support visual review, but they do not replace interaction evidence or the production-gap audit.

## Legitimate hybrid seams

A hybrid should name complete components, not visual traits such as `use the spacing from B`.

- `A owner ledger + B public timeline + B shared update + A detail sheet` is structurally coherent because all four can consume the same projection while assigning different reading priorities.
- `A owner interaction grammar + C public horizon + B shared update + C detail inspector` preserves explicit owner controls, spatial public orientation, editorial forwarding, and horizon context.
- `B purpose header and change margin inside A owner ledger` is a bounded composition change if the ledger remains the primary owner structure.
- `C current-position marker inside A public ledger` may be used only as one earned emphasis. Importing C's full weighted horizon would make it a C public surface, not an A styling detail.

Avoid combining two dominant structures on one surface. A ledger plus a chapter index plus a horizon rail would recreate the hierarchy problem the redesign is meant to solve.

## Gate to Phase 2

Phase 2 begins only after an explicit selection. The selection approves a direction for production implementation and verification. It does not waive the unresolved production work around data convergence, durable publication, privacy projection, anonymous access, source tracking, responsive behaviour, accessibility, or migration safety.
